## 프로젝트 소개

<div style="display:flex; flex-wrap: wrap;">
  <img src="https://user-images.githubusercontent.com/37692675/111239812-4e512580-863d-11eb-98b3-e342534ce25b.png" alt="스크린샷1" width="49%" />
  <img src="https://user-images.githubusercontent.com/37692675/111240029-c3bcf600-863d-11eb-8d92-b5bd3baf242f.png" alt="스크린샷1" width="49%" />
</div>
<br />

디자인 시스템이란 디자이너, 개발자, 기획자 등 관련 이해 관계자들이 프로젝트를 일관성이 있게 구축할 수 있도록 도와주는 모든 것들을 포함합니다.  

<b>디자인 시스템의 장점</b><br />
<b>1) 코드의 일관성</b><br />
만들어 둔 컴포넌트를 공용으로 가져다가 사용하기 때문에 여러 사람이 작업하여도 크게 벗어나지 않는 선에서 작업하기 때문에 코드가 일관성이 생깁니다.

<b>2) 빠른 구축</b><br />
초기 디자인 시스템을 구성하는 것은 어렵고 오래 걸릴 수도 있지만, 만들어두기만 하면 신규 페이지가 들어갈 때 빠르게 만들 수 있다.

<b>3) 중복 코드 남용/덮어쓰기 방지</b><br />
예전에 작업했을 때, 분명 다른 페이지에서 똑같은 CSS가 사용됨에도 불구하고 부모 class가 달라 코드를 똑같은 코드를 재작성하거나 중첩을 계속 쌓으면서 덮어쓰는 그러한 현상이 발생했었습니다. <br />디자인 시스템으로 가이드라인을 확실하게 잡은 뒤, 디자인도 일관성 있고 코드도 훨씬 깨끗하게 사용될 수 있습니다.
<br />
<br />

디자인 시스템을 처음부터 구축하려면 막막하게 됩니다. 그래서 이러한 상황을 방지하기 위해 미리 만들어둔 베이스를 이용해 쉽고 빠르게 제작할 수 있습니다.
